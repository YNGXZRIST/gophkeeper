# GophKeeper

Менеджер секретов с клиент-серверной архитектурой. Хранит логины/пароли, произвольный текст, бинарные файлы и банковские карты — к каждой записи можно добавить текстовую метаинформацию (сайт, банк, заметку и т.п.). Клиент работает в терминале под Windows, Linux и macOS; сервер хранит данные и синхронизирует их между всеми устройствами одного владельца. Обмен идёт по gRPC поверх обязательного TLS.

## Что умеет

- регистрация, аутентификация и авторизация пользователей (JWT: access + refresh);
- хранение четырёх типов данных с произвольной метаинформацией: логин/пароль, текст, бинарные файлы, банковские карты;
- синхронизация данных между несколькими клиентами одного владельца;
- терминальный интерфейс (TUI);
- информация о версии и дате сборки клиента (`client --version`).

## Как это устроено

```
┌─────────────────────┐   gRPC + TLS    ┌─────────────────┐      ┌────────────┐
│   Клиент (TUI)      │ ──────────────► │     Сервер      │ ───► │ PostgreSQL │
│  локальный SQLite   │ ◄────────────── │                 │      └────────────┘
└─────────────────────┘  синхронизация  └─────────────────┘
```

- **Клиент** — интерактивный TUI (bubbletea), держит локальный кэш в SQLite и синхронизирует его с сервером.
- **Сервер** — gRPC-сервис поверх PostgreSQL.
- **TLS обязателен** с обеих сторон. Сертификат self-signed: публичный вшит в клиент через `go:embed`, приватный ключ остаётся только на сервере и в git не попадает. Генерируется утилитой `cmd/certgen` (`make certs`).

## Быстрый старт

Нужны **Go 1.26+** и **Docker** (под PostgreSQL).

```bash
make db-up        # поднять PostgreSQL в Docker
make run-server   # запустить сервер (сам сгенерит сертификаты и поднимет БД)
make run-client   # в другом терминале — клиент
```

Сервер читает конфигурацию из JSON, переменных окружения и флагов (приоритет: флаги > env > JSON > дефолты). Секреты `JWT_SECRET` и `REFRESH_SECRET` обязательны. Пример конфига — [`examples/server.config.example.json`](examples/server.config.example.json):

```bash
go run ./cmd/server -c examples/server.config.example.json
```

DSN под локальный `make db-up`: `postgres://gophkeeper:gophkeeper@localhost:5432/gophkeeper?sslmode=disable`.

## Сборка клиента

```bash
make build             # сервер и клиент под текущую платформу → bin/
make build-client-all  # клиент под все платформы сразу:
                       #   bin/client-windows-amd64.exe
                       #   bin/client-linux-amd64
                       #   bin/client-darwin-amd64
                       #   bin/client-darwin-arm64
```

Адрес сервера вшивается на сборке: `GRPC_ADDR=host:port make build-client` (по умолчанию `localhost:8080`).

```bash
./bin/client --version   # версия и дата сборки
```

## Тестирование

```bash
make test              # юнит-тесты (-race) + покрытие
make test-integration  # интеграционные тесты (нужен Docker)
make check             # формат + go vet + statictest + linter + тесты
```
